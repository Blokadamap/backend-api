в данный момент всё происходит локально поэтому ожидается, что апи базы данных висит на `127.0.0.1:8080`
смотри https://github.com/Blokadamap/database-core

## API

На данный момент реализованы запросы к базе данных разных данных об авторе через author_id
и получение family_status_name через family_status_id

Пример URL: `/api/v1/author/social_classes/1`

На данный момент это делается через получение автора по id и последующего извлечения нужных полей, 
что неэффективно, с учётом того, что у нас есть отдельные таблицы вида `author_to_*` для такого,
но так как для них сейчас нет ручек со стороны бд...

То что просили эндпоинты для "маркеров" - `/api/v1/note/detailed/{id}`. `id` - это `note_id` для которой возвращается подробная информация, а именно сама note, её автор, и список `point` которые к этому событию относятся.

## Loader

### Загрузка из файла
Можно загружать в базу данных из файла

Утилита `loader.py`

Там человеческий `--help`, за подробностями туда. Вкраце, принимает json,
с описанием загружаемых авторов, заметок(и тегов) и точек. Больше ничего пока не умеем загружать, 
потому что нет соотвествуюещего апи у бд

Дли примера
```
python loader.py .\load_example.json
```

Для него есть API по URL: `/api/v1/loader/load`
